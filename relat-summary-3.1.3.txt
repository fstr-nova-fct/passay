  @Override
  public RuleResult validate(final PasswordData passwordData)
  {
    PassayUtils.assertNotNullArg(passwordData, "Password data cannot be null");
    if (numCharacteristics > rules.size()) { throw new IllegalStateException(...); }

    int successCount = 0;
    final List<RuleResultDetail> details = new ArrayList<>();
    final List<RuleResultMetadata> metadata = new ArrayList<>();
    for (CharacterRule rule : rules) {
      final RuleResult rr = rule.validate(passwordData);
      if (rr.isValid()) { successCount++; }
      if (reportRuleFailures) { details.addAll(rr.getDetails()); }
      metadata.add(rr.getMetadata());
    }
    final boolean valid = successCount >= numCharacteristics;
    if (!valid && reportFailure) {
      details.add(new RuleResultDetail(ERROR_CODE, createRuleResultDetailParameters(successCount)));
    }
    return valid ? new SuccessRuleResult(...) : new FailureRuleResult(..., details);
  }

Why CharacterCharacteristicsRule
Needs at least N of M character-type rules to pass (here N=2, M=3)
Each sub-rule is a clear yes/no cause (uppercase, digit, and special)
The final pass/fail depends on how many causes are true —> good for cause-effect graph and a full decision table (2^3=8)

Specification:
Rule: CharacterCharacteristicsRule(2, upper, digit, specialAscii) — at least 2 of 3 (N=2, M=3)
->C1: password has ≥1 uppercase (CharacterRule UpperCase)
->C2: password has ≥1 digit (CharacterRule Digit)
->C3: password has ≥1 special ASCII (CharacterRule SpecialAscii)
  (SpecialAscii not Special: ASCII passwords only; Passay's default test uses Special = wider charset)

Cause-effect graph
Causes: C1, C2, C3 (each sub-rule validates)
Intermediate: successCount = number of causes C1, C2, C3 that are true
Condition: password valid when successCount ≥ 2 (numCharacteristics = 2 in code)

Effects:
E1 — password accepted (SuccessRuleResult)
E2 — password rejected (INSUFFICIENT_CHARACTERISTICS)

Links:
each of C1, C2, C3 true → adds 1 to successCount
successCount ≥ 2 → E1
successCount < 2 → E2
(C1/C2/C3 do not link to E1/E2 directly — only through the count)
